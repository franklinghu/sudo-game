# Product Requirements Document: AI 智能生产车间排产系统

**版本**: 1.0
**日期**: 2026-03-18
**作者**: Sarah (Product Owner)

## 执行摘要

本系统旨在为离散制造车间（机械零部件、自动化设备）打造一套 AI 驱动的智能排产解决方案，解决当前订单延期频繁（20%）、紧急插单难以处理、人工排产效率低（3 小时/次）的核心痛点。

通过集成金蝶 ERP、自研 MES 及 IoT 设备数据，系统将实现智能排程优化和动态调整两大核心能力，目标将排产时间从 3 小时缩短至 10 分钟，设备利用率从 70% 提升至 90%，订单延期率从 20% 降至 5%。

首期试点 3-5 条生产线，验证核心算法后逐步推广至全部 30 条生产线。

## 问题陈述

**当前状况**: 
- 人工排产耗时 3 小时/次，效率低下
- 订单延期率高达 20%，影响客户满意度
- 紧急插单无法快速响应，打乱原有生产计划
- 设备利用率仅 70%，存在产能浪费
- 换线时间因产品组合不同差异大，缺乏优化

**拟议解决方案**: 
构建 AI 智能排产系统，基于历史数据和实时状态，自动优化生产顺序，最小化换线时间，支持动态调整和紧急插单处理。

**业务影响**: 
- 排产效率提升 18 倍（3 小时 → 10 分钟）
- 设备利用率提升 20 个百分点（70% → 90%）
- 订单延期率降低 75%（20% → 5%）

### 主要用户：生产计划员
- **角色**: 负责日常排产、调整计划、处理插单
- **目标**: 快速生成最优排产计划，减少加班，降低延期
- **痛点**: 人工排产耗时长、考虑因素多易出错、插单时手忙脚乱

## 用户故事与验收标准

### 故事 1：智能排程生成

**作为** 生产计划员
**我想要** 一键生成未来 7 天的最优排产计划
**以便于** 快速完成排产工作，减少人工计算时间

**验收标准:**
- [ ] 系统能在 10 分钟内完成 30 条生产线、1000 订单的排产计算
- [ ] 排产结果考虑换线时间优化（同系列产品优先连续生产）
- [ ] 排产结果考虑工序先后约束
- [ ] 排产结果考虑设备/产线特殊要求
- [ ] 排产结果可导出为 Excel 或直接下发至 MES

### 故事 2：紧急插单处理

**作为** 生产计划员
**我想要** 快速处理紧急插单并查看影响分析
**以便于** 在不影响已执行订单的前提下响应客户需求

**验收标准:**
- [ ] 插单录入后 5 分钟内完成重排计算
- [ ] 系统自动识别可调整的订单（未下发、未执行）
- [ ] 系统锁定已下发且正在执行的订单，不允许调整
- [ ] 显示插单对其他订单交期的影响
- [ ] 支持按客户等级、交期紧急程度定义优先级

### 故事 3：动态调整与异常响应

**作为** 生产计划员
**我想要** 系统自动响应设备故障等异常情况并重新排产
**以便于** 快速恢复生产秩序，减少停机损失

**验收标准:**
- [ ] 系统接收 IoT 设备故障信号后自动触发重排
- [ ] 重排时自动排除故障设备，将任务分配至替代产线
- [ ] 通知相关人员（生产计划员、车间主任）
- [ ] 记录异常事件及处理结果

### 故事 4：排产结果可视化

**作为** 管理层
**我想要** 通过可视化看板查看排产计划和生产进度
**以便于** 快速掌握生产状态，做出决策

**验收标准:**
- [ ] 提供甘特图视图展示各产线排产计划
- [ ] 支持按日期、产线、产品等多维度筛选
- [ ] 实时显示计划执行进度（已完成/进行中/待开始）
- [ ] 延期订单高亮预警
- [ ] 支持移动端查看

### 故事 5：系统集成与数据同步

**作为** 生产计划员
**我想要** 系统自动从金蝶 ERP 和 MES 同步数据
**以便于** 无需手动录入，保证数据准确性

**验收标准:**
- [ ] 自动从金蝶 ERP 同步销售订单、物料信息
- [ ] 自动从 MES 同步工序信息、工艺路线
- [ ] 自动从 IoT 数据库同步设备状态、实时进度
- [ ] 数据同步延迟 ≤5 分钟
- [ ] 支持手动触发同步

## 功能需求

### 核心功能

#### 功能 1：智能排程引擎
- **描述**: 基于 AI 算法的生产排程优化引擎，考虑多约束条件自动生成最优排产计划
- **用户流程**: 
  1. 计划员点击"生成排产计划"
  2. 系统读取待排产订单、产线状态、工艺约束
  3. AI 引擎计算最优排产方案
  4. 展示排产结果（甘特图 + 列表）
  5. 计划员确认后下发至 MES
- **边缘情况**: 
  - 订单量超过产能时，按优先级排序并提示产能不足
  - 关键物料短缺时，自动延后相关订单
- **错误处理**: 计算超时（>15 分钟）时提示并建议分批排产

#### 功能 2：换线时间优化
- **描述**: 基于产品相似度自动优化生产顺序，最小化换线时间
- **用户流程**: 
  1. 系统读取产品换线矩阵（A→B、A→C 等换线时间）
  2. 排产时优先安排换线时间短的产品组合
  3. 在排产结果中展示预计换线时间
- **边缘情况**: 
  - 新产品无换线数据时，使用默认值或相似产品参考
- **错误处理**: 换线矩阵数据缺失时告警

#### 功能 3：紧急插单处理
- **描述**: 快速处理插单请求，自动分析影响并重新排产
- **用户流程**: 
  1. 计划员录入插单信息（产品、数量、交期、优先级）
  2. 系统分析可调整的订单范围
  3. 执行局部重排
  4. 展示影响分析（哪些订单延期、延期多久）
  5. 计划员确认后执行
- **边缘情况**: 
  - 插单导致多个订单延期时，需计划员确认
  - 无可用产能时，建议外包或加班方案
- **错误处理**: 重排失败时回滚并提示原因

#### 功能 4：动态调整引擎
- **描述**: 实时响应设备故障、物料短缺等异常，自动重新排产
- **用户流程**: 
  1. 系统接收异常信号（IoT 设备故障、MES 报工异常）
  2. 自动触发重排引擎
  3. 将受影响任务重新分配至可用产线
  4. 通知相关人员
- **边缘情况**: 
  - 多设备同时故障时，按优先级处理
  - 无替代产线时，通知计划员人工介入
- **错误处理**: 重排失败时发送告警通知

#### 功能 5：可视化看板
- **描述**: 提供排产计划、执行进度、异常预警的可视化展示
- **用户流程**: 
  1. 用户打开看板页面
  2. 选择查看维度（产线、日期、产品）
  3. 查看甘特图、进度条、预警信息
  4. 支持钻取查看详情
- **边缘情况**: 
  - 数据加载超时时显示缓存数据并提示
- **错误处理**: 数据异常时显示占位符并提示刷新

### 不在范围内

- 供应链协同排产（未来考虑）
- 移动端排产操作（首期仅支持查看）

## 技术约束

### 性能
- **排产计算时间**: ≤10 分钟（1000 订单，30 产线）
- **插单重排时间**: ≤5 分钟
- **数据同步延迟**: ≤5 分钟
- **并发用户**: 支持 10 个计划员同时操作
- **系统可用性**: ≥99%（工作时间）

### 安全
- **认证**: 与企业现有 AD/LDAP 集成
- **授权**: 基于角色的访问控制（RBAC）
- **数据保护**: 生产数据加密存储，传输使用 HTTPS
- **审计日志**: 记录所有排产操作和调整记录
- **合规**: 符合企业信息安全规范

### 集成
- **金蝶 ERP**: 通过 API 或数据库视图同步订单、物料数据
- **自研 MES**: 通过 API 同步工序、工艺路线、报工数据
- **IoT 数据库**: 读取设备状态、实时进度数据
- **通知系统**: 与企业微信/钉钉集成，发送预警通知

### 技术栈
- **后端**: Python/Java（推荐 Python，AI 生态丰富）
- **AI 引擎**: 运筹优化算法（遗传算法、模拟退火、约束规划）
- **数据库**: PostgreSQL（排产数据）+ Redis（缓存）
- **前端**: Vue.js/React + ECharts（可视化）
- **部署**: Docker 容器化，支持 Kubernetes
- **兼容性**: Chrome/Edge 浏览器，支持 1920x1080 分辨率

## 依赖与阻碍

**依赖项:**
- **金蝶 ERP 接口**: 需要 ERP 团队提供订单、物料数据接口（负责人：ERP 团队）
- **MES 系统改造**: 需要 MES 团队支持排产结果下发接口（负责人：MES 团队）
- **IoT 数据接入**: 需要设备团队提供设备状态数据（负责人：设备科）
- **服务器资源**: 需要 IT 部门提供部署服务器（负责人：IT 部门）

**已知阻碍:**
- **数据质量**: 历史换线时间数据可能不完整，需要补充采集（解决方案：首期使用估算值，逐步完善）
- **产线差异**: 30 条产线设备能力不同，需要详细调研（解决方案：首期试点 3-5 条典型产线）
