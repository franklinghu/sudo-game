## 3. 服务详细设计

### 3.1 排产服务 (Scheduling Service)

**职责**: 核心排产逻辑，调用 AI 引擎生成最优排产计划

**技术栈**: Python + FastAPI + Celery

**API 端点**:
```python
POST   /api/v1/scheduling/generate      # 生成排产计划
POST   /api/v1/scheduling/rush-order    # 插单重排
POST   /api/v1/scheduling/adjust        # 手动调整
GET    /api/v1/scheduling/result/{id}   # 获取排产结果
GET    /api/v1/scheduling/status/{id}   # 查询计算状态
```

**核心流程**:
```
1. 接收排产请求 (订单列表、产线信息、约束条件)
   ↓
2. 数据预处理 (验证、格式化、缓存)
   ↓
3. 调用 AI 排程引擎 (异步任务)
   ↓
4. 轮询/回调获取结果
   ↓
5. 结果后处理 (格式化、存储)
   ↓
6. 返回排产计划
```

**数据库表设计**:
```sql
-- 排产任务表
CREATE TABLE scheduling_tasks (
    id              UUID PRIMARY KEY,
    task_type       VARCHAR(50),      -- 'regular' | 'rush' | 'adjust'
    status          VARCHAR(20),      -- 'pending' | 'running' | 'completed' | 'failed'
    request_data    JSONB,
    result_data     JSONB,
    created_by      UUID,
    created_at      TIMESTAMP DEFAULT NOW(),
    completed_at    TIMESTAMP
);

-- 排产计划表
CREATE TABLE production_plans (
    id              UUID PRIMARY KEY,
    plan_date       DATE,
    version         INTEGER,
    is_active       BOOLEAN,
    plan_data       JSONB,            -- 完整排产计划
    metrics         JSONB,            -- KPI 指标
    created_at      TIMESTAMP DEFAULT NOW()
);

-- 排产任务 - 产线关联表
CREATE TABLE plan_line_assignments (
    id              UUID PRIMARY KEY,
    plan_id         UUID REFERENCES production_plans(id),
    line_id         VARCHAR(50),
    order_id        VARCHAR(50),
    start_time      TIMESTAMP,
    end_time        TIMESTAMP,
    changeover_time INTEGER,          -- 换线时间 (分钟)
    sequence        INTEGER           -- 生产顺序
);
```

## 6. 错误处理与重试

### 错误分类
| 错误类型 | 处理策略 | 重试次数 |
|----------|----------|----------|
| 网络超时 | 指数退避重试 | 3 次 |
| 数据库死锁 | 立即重试 | 2 次 |
| AI 引擎超时 | 降级 (使用上次的排产方案) | 1 次 |
| 外部系统不可用 | 告警 + 人工介入 | 0 次 |

## 10. 关键决策与权衡

### 决策 1: 单体 vs 微服务
**选择**: 试点阶段单体，生产阶段微服务

**理由**:
- ✅ 试点阶段团队小、迭代快
- ✅ 减少运维复杂度
- ⚠️ 后期需要重构为微服务

### 决策 2: 精确算法 vs 启发式算法
**选择**: 混合策略

**理由**:
- 小规模 (<100 订单): 使用 CP-SAT 精确求解
- 大规模 (>100 订单): 使用遗传算法 + 局部优化
- 平衡求解质量和时间

### 决策 3: 实时同步 vs 定时同步
**选择**: 混合策略

**理由**:
- ERP 订单：定时同步 (5 分钟)
- MES 报工：实时同步 (WebSocket)
- IoT 设备：实时订阅 (MQTT)
